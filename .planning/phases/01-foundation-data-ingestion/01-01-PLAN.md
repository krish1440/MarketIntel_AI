# 01-01-PLAN: Local PostgreSQL Setup and Schema Initialization

This plan establishes the persistent storage layer for the system using a Dockerized PostgreSQL instance and initializes the database schema.

## Tasks

<task>
<read_first>
- .planning/research/STACK.md
- .planning/phases/01-foundation-data-ingestion/01-CONTEXT.md
</read_first>
<action>
Create a `docker-compose.yml` file in the project root to spin up a PostgreSQL 16 container.
- Image: `postgres:16-alpine`
- Environment: `POSTGRES_DB=stock_intelligence`, `POSTGRES_USER=admin`, `POSTGRES_PASSWORD=password`
- Ports: `5432:5432`
- Volumes: `./pgdata:/var/lib/postgresql/data`
</action>
<acceptance_criteria>
- `docker-compose.yml` exists and contains the correct service definition.
- `pgdata` directory is listed in `.gitignore`.
</acceptance_criteria>
</task>

<task>
<read_first>
- docker-compose.yml
</read_first>
<action>
Create `db/init.sql` with the initial schema definition.
- Table `stocks`: `id SERIAL PRIMARY KEY`, `ticker VARCHAR(20) UNIQUE NOT NULL`, `name VARCHAR(100)`, `exchange VARCHAR(10)`.
- Table `live_quotes`: `id SERIAL PRIMARY KEY`, `stock_id INTEGER REFERENCES stocks(id)`, `price DECIMAL(15, 2)`, `change_percent DECIMAL(10, 4)`, `volume BIGINT`, `timestamp TIMESTAMP DEFAULT CURRENT_TIMESTAMP`.
- Index on `live_quotes(stock_id, timestamp)`.
</action>
<acceptance_criteria>
- `db/init.sql` exists and contains the SQL schema.
- Foreign key relationship between `live_quotes` and `stocks` is defined.
</acceptance_criteria>
</task>

<task>
<read_first>
- db/init.sql
</read_first>
<action>
Create `db/schema.py` using SQLAlchemy to map the database tables to Python objects.
- Define `Base`, `Stock`, and `LiveQuote` classes matching the SQL schema.
- Provide a `get_engine()` function using the credentials from `docker-compose.yml`.
</action>
<acceptance_criteria>
- `db/schema.py` exists and correctly maps the SQL fields.
- `db/schema.py` contains the database connection string.
</acceptance_criteria>
</task>

## Verification Criteria
1. `docker compose up -d` successfully starts the database.
2. PostgreSQL is accessible on port 5432.
3. Tables `stocks` and `live_quotes` are present in the `stock_intelligence` database.

## must_haves
- [ ] Database is persistent and accessible locally.
- [ ] Schema supports both metadata and time-series quotes.
- [ ] SQLAlchemy models are ready for use by ingestion scripts.
