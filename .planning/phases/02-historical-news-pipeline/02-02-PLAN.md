# 02-02-PLAN: News Ingestion Engine

This plan implements the service to aggregate financial news related to the stocks being tracked.

## Tasks

<task>
<read_first>
- db/init.sql
- db/schema.py
</read_first>
<action>
Update `db/init.sql` and `db/schema.py` to include the `news_articles` table.
- Table `news_articles`: `id SERIAL PRIMARY KEY`, `stock_id INTEGER REFERENCES stocks(id)`, `title TEXT`, `summary TEXT`, `url TEXT UNIQUE`, `published_at TIMESTAMP`, `sentiment_score DECIMAL(10, 4) DEFAULT NULL`.
</action>
<acceptance_criteria>
- `news_articles` table is defined in SQL and SQLAlchemy.
</acceptance_criteria>
</task>

<task>
<read_first>
- requirements.txt
</read_first>

</action>
<acceptance_criteria>
- `requirements.txt` is updated.
</acceptance_criteria>
</task>

<task>
<read_first>
- db/schema.py
</read_first>
<action>
Create `ingestion/news_aggregator.py`.
- Use `feedparser` to fetch news from RSS feeds (e.g., Google News for stock-specific keywords).
- Extract headlines, summaries, and timestamps.
- Map articles to `stock_id` based on keywords in the title/summary.
- Store articles in the `news_articles` table, avoiding duplicates (using URL).
</action>
<acceptance_criteria>
- `ingestion/news_aggregator.py` exists and runs.
- At least 10 news articles are successfully stored in the database for the tracked stocks.
</acceptance_criteria>
</task>

## Verification Criteria
1. `news_articles` table contains entries with valid stock IDs.
2. No duplicate articles (verified by URL).
3. The `published_at` timestamps are correctly parsed and stored.

## must_haves
- [ ] Schema supports news articles with a placeholder for sentiment.
- [ ] Aggregator correctly identifies stocks from news text.
- [ ] Duplicates are handled via URL uniqueness.
