# 02-01-PLAN: Historical Data Backfill

This plan implements the logic to fetch and store historical daily price data for all stocks in the database.

## Tasks

<task>
<read_first>
- db/init.sql
- db/schema.py
</read_first>
<action>
Update `db/init.sql` and `db/schema.py` to include the `historical_prices` table.
- Table `historical_prices`: `id SERIAL PRIMARY KEY`, `stock_id INTEGER REFERENCES stocks(id)`, `date DATE NOT NULL`, `open DECIMAL(15, 2)`, `high DECIMAL(15, 2)`, `low DECIMAL(15, 2)`, `close DECIMAL(15, 2)`, `volume BIGINT`.
- Unique constraint on `(stock_id, date)`.
</action>
<acceptance_criteria>
- `historical_prices` table is defined in SQL and SQLAlchemy.
- Migration script or manual update applied to local DB.
</acceptance_criteria>
</task>

<task>
<read_first>
- ingestion/poll_prices.py
</read_first>
<action>
Create `ingestion/backfill_history.py`.
- Function `get_yahoo_ticker(symbol, exchange)`: Maps NSE to `.NS` and BSE to `.BO`.
- Use `yfinance` to download 2 years of daily data for all stocks in the `stocks` table.
- Upsert the data into the `historical_prices` table.
- Handle rate limiting and symbol mapping errors.
</action>
<acceptance_criteria>
- `ingestion/backfill_history.py` exists.
- Successfully downloads data for at least 5 stocks as a test.
- Data is correctly stored in the `historical_prices` table.
</acceptance_criteria>
</task>

## Verification Criteria
1. `historical_prices` table contains data for the seeded stocks.
2. The `date` column has a continuous range of daily data for the past 2 years.
3. No duplicate entries for the same stock/date.

## must_haves
- [ ] Schema supports historical EOD data.
- [ ] Backfill script handles symbol mapping for NSE/BSE.
- [ ] Data is stored efficiently with unique constraints.
