## 📖 1. The Core Philosophy
MarketIntel AI is not just a stock scraper; it is a **Neural Financial Gateway**. The API is designed to solve the "Latency of Intelligence" problem—the gap between raw market movements and actionable insight. 

We bridge three distinct worlds:
1.  **The Database (SQLAlchemy):** Stable, relational storage for 4.5M+ records.
2.  **The Real-World (YFinance/RSS):** High-velocity, raw data streams.
3.  **The Neural Stack (Transformers/LSTM):** Where raw numbers turn into sentiment and predictions.

---

## 🏗️ 2. Architectural Blueprint
The API follows a **Non-Blocking Orchestration** pattern. Instead of waiting for massive data syncs to finish, the API serves cached "Neural Snapshots" while background workers (`delta_update.py`) patch the gaps in the dark.

## 📡 3. Detailed Endpoint Intelligence (The API Catalog)

### **A. Discovery Layer**
#### `GET /api/stocks`
*   **Purpose:** The primary data-feed for the "Market Explorer" grid.
*   **Logic:** Executes a fuzzy search across the `Stock` metadata table and performs a **Left Join** on the `LiveQuote` table to attach the most recent pricing.
*   **Parameters:** `page` (int), `limit` (int), `search` (string).
*   **Neural Impact:** Acts as the entry point for users to identify high-interest tickers for further AI analysis.

### **B. Predictive Layer**
#### `GET /api/predict/{ticker}`
*   **Purpose:** Triggers the LSTM Neural Forecasting engine.
*   **Logic:** Pulls the last 100 trading sessions, normalizes the data, and passes it through the trained LSTM weights.
*   **Response:** Returns a `signal` (BUY/SELL/HOLD), `target_price`, and `confidence_score`.
*   **Architecture Note:** Uses a Singleton pattern for the `PredictionService` to avoid expensive model reloading.

#### `GET /api/model-status`
*   **Purpose:** Monitors the "Neural Health" of the system.
*   **Logic:** Reads from `models/checkpoints/metadata.json`.
*   **Data Return:** Last training timestamp and RMSE (accuracy) metrics.

### **C. Sentiment Layer**
#### `GET /api/news/{ticker}`
*   **Purpose:** Provides a human-readable context for price movements.
*   **Logic:** Fetches the top 10 most recent articles from the `NewsArticle` table.
*   **Neural Impact:** Each article includes a `sentiment` score generated by the DistilBERT Transformers model.

#### `POST /api/news/refresh/{ticker}`
*   **Purpose:** Manual override for real-time intelligence.
*   **Logic:** Forces an immediate RSS scrape and AI sentiment analysis.
*   **Architecture Note:** Crucial for tracking "breaking news" that hasn't been captured by the 10-minute background sync.

### **D. Historical Layer**
#### `GET /api/history/{ticker}`
*   **Purpose:** High-performance charting data.
*   **Logic:** Retrieves a time-series array of closing prices for the last `X` days.
*   **Integrity Guard:** Uses the `clean_nas` filter to ensure 100% JSON compliance for charting libraries.

## 🛡️ 4. Resilience & Reliability (The "Safety Valves")

### **The NaN Guard (`clean_nas`)**
Financial data is messy. APIs often crash on `NaN` (Not a Number) values from broken feed data. Our API uses a recursive `clean_nas` filter that sanitizes every JSON response, ensuring the frontend never sees a "Hydration Error" or "Value Type" crash.

### **Neural Model Caching**
The `PredictionService` and `SentimentExpert` follow a **Singleton Pattern**. We don't reload the models for every request; we keep them hot in memory, drastically reducing the "Time to First Insight."
